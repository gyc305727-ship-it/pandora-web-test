# 智能掌-上工作系统 – 前端阶段性进展报告

> 更新时间：2025-10-10

## 一、项目定位回顾
Web 管理后台仅负责 **用户角色与权限管理**，为移动端提供账号生命周期维护功能；其余模块（任务、日志、AI 等）由安卓 App 实现。

## 二、本阶段已完成工作
1. **基础框架搭建**
   - 引入 Vue 3 + Element-Plus + Pinia。
   - 全局布局 `Layout`（侧边栏 / 顶栏 / 面包屑）与懒加载路由完成。
2. **身份认证**
   - 管理员登录页（本地模拟 Token）。
   - 全局前置守卫：未登录重定向、已登录阻止回到登录页。
3. **用户管理模块**
   - 表格展示用户列表，支持姓名 / 邮箱 / 角色筛选。
   - 统计卡片：用户总数、活跃、禁用数量。
   - 新增 / 编辑 / 删除用户（含表单校验）。
   - 业务规则：唯一“创始人”保护；创始人/系统管理员禁止删改。
   - 本地日志（Pinia store）记录增删改操作。
4. **技术基础设施**
   - Axios 二次封装，预留请求/响应拦截器。
   - Vite 持续集成脚手架。

## 三、当前遗留问题 / 待办事项
| 分类 | 问题描述 |
| :-- | :-- |
| 账户生命周期 | 缺少退出登录、Token 过期刷新逻辑 |
| 数据交互 | 用户列表仍为全量拉取，分页未真正落地；筛选应调用后端接口而非前端过滤 |
| 权限体系 | 角色作为字段存在，但未实现动态路由 / 按钮级权限控制 |
| 公司十大事项 | 模块接口 404，UI 未正常增删改；无需优先级字段需调整 |
| 质量保障 | 缺少 ESLint / Prettier 规范与单元测试框架 |

## 四、下一阶段计划（MVP 完成）
1. 实现退出登录与 401 拦截；抽象 `useAuth` Hook。
2. 与后端对齐分页 / 查询接口，落地真实分页。
3. 修复“公司十大事项”增删改查，保持仅展示列表（无优先级）。
4. 抽象角色-权限模块：在用户管理内部配置角色，下发动态菜单权限。
5. 引入 ESLint+Prettier+Husky，配置 Vitest 单测。
