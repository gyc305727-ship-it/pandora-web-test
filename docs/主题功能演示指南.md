---

cd web

- 用户名：`admin`
- 密码：（根据实际情况）

## 🎯 演示流程（建议 3-5 分钟）

### 第一步：介绍功能背景（30秒）

**演示词**：
> "为了提升用户体验，我们为系统增加了全局主题切换功能。用户可以根据个人偏好或使用环境，在浅色和暗色主题之间自由切换。这个功能在现代 Web 应用中非常常见，比如 GitHub、VS Code 等都提供了类似功能。"

---

### 第二步：展示切换入口（30秒）

**操作**：
1. 登录后，指向顶部导航栏右上角的月亮图标
2. 鼠标悬停在图标上，展示 Tooltip 提示

**演示词**：
> "主题切换按钮位于顶部导航栏右上角，用户下拉菜单的左侧。我们使用了直观的图标设计：月亮图标表示当前是浅色模式，可以切换到暗色；太阳图标表示当前是暗色模式，可以切换到浅色。鼠标悬停时会显示提示信息。"

---

### 第三步：演示主题切换（1分钟）

**操作**：
1. 点击月亮图标，切换到暗色模式
2. 等待 1-2 秒，让老师观察变化
3. 指出几个关键变化区域：
   - 整体背景变深
   - 侧边栏颜色变化
   - 卡片背景变化
   - 文字颜色自动调整为浅色
4. 再次点击（现在是太阳图标），切换回浅色模式

**演示词**：
> "点击图标后，整个系统会平滑过渡到暗色模式。您可以看到：
> - 整体背景从浅灰色变为深黑色
> - 侧边栏也相应变暗
> - 所有卡片和内容区域都适配了暗色背景
> - 文字颜色自动调整为浅色，确保可读性
> - 切换过程有平滑的渐变效果，不会突兀
> 
> 我们再切换回浅色模式，可以看到所有元素又恢复到原来的浅色风格。"

---

### 第四步：展示不同页面的适配（1分钟）

**操作**：
1. 切换到暗色模式
2. 依次访问以下页面，展示适配效果：
   - **仪表盘**：统计卡片、日志时间线
   - **用户管理**：表格、搜索框、按钮
   - **公司十大事项**：事项卡片

**演示词**：
> "主题切换是全局生效的，我们来看看不同页面的适配效果：
> 
> **仪表盘页面**：统计卡片、图表、日志时间线都完美适配了暗色主题。
> 
> **用户管理页面**：表格、搜索框、按钮等所有 Element Plus 组件都跟随主题变化。
> 
> **公司十大事项页面**：事项卡片在暗色模式下依然清晰易读。
> 
> 所有页面的组件都经过了适配，确保在暗色模式下也有良好的视觉体验。"

---

### 第五步：演示持久化功能（30秒）

**操作**：
1. 确保当前是暗色模式
2. 按 F5 刷新页面
3. 观察页面加载后依然是暗色模式

**演示词**：
> "用户的主题选择会被保存到浏览器本地存储中。现在我刷新页面，您可以看到系统依然保持暗色模式，而不是回到默认的浅色。这样用户就不需要每次访问都重新设置主题了。"

---

### 第六步：技术亮点说明（1分钟）

**演示词**：
> "这个功能的技术实现有几个亮点：
> 
> 1. **使用 Element Plus 官方暗色模式**：我们采用了 Element Plus 官方提供的暗色主题 CSS 变量，确保所有组件都能完美适配。
> 
> 2. **CSS 变量方案**：通过定义 CSS 变量来控制颜色，切换时只需要改变一个类名，性能非常高。
> 
> 3. **状态管理**：使用项目已有的 Pinia 状态管理库，保持技术栈一致。
> 
> 4. **系统偏好检测**：首次访问时会自动检测用户的操作系统主题偏好，如果系统是暗色模式，我们也会默认使用暗色。
> 
> 5. **最小改动原则**：整个功能只涉及 5 个文件的改动，约 300 行代码，不影响任何现有业务逻辑。"

## 💡 可能的提问与回答

### Q1: 为什么要做这个功能？
**A**: 
- 提升用户体验，满足不同用户的偏好
- 暗色模式在夜间或低光环境下更护眼
- 这是现代 Web 应用的标配功能
- 展示我们对细节和用户体验的重视

### Q2: 实现难度大吗？
**A**: 
- 技术难度适中，主要是 CSS 变量和状态管理
- 使用了 Element Plus 官方暗色模式，减少了适配工作量
- 整个实现约 300 行代码，5 个文件改动
- 开发时间约 2-3 小时

### Q3: 会影响现有功能吗？
**A**: 
- 完全不会，这是纯 UI 层的改动
- 所有业务逻辑保持不变
- 已经过充分测试，无副作用

### Q4: 后续可以扩展吗？
**A**: 
- 可以增加更多主题色（蓝色、绿色等）
- 可以根据时间自动切换主题
- 可以让用户自定义主题色值
- 架构设计上已经考虑了扩展性
